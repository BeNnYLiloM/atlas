Техническое Задание (ТЗ) на разработку корпоративного мессенджера "Atlas"

Версия документа: 1.0 (MVP)
Дата: 26.11.2025
Статус: Черновик / Утверждение

1. Паспорт Проекта

1.1. Название и Суть

Проект "Atlas" — корпоративная платформа для коммуникаций (Team Collaboration Platform), объединяющая возможности текстовых чатов, управления задачами и видеоконференцсвязи высокого качества.

1.2. Проблема и Решение

Проблема: Блокировка и риски использования зарубежных сервисов (Slack, Discord, MS Teams) в РФ. Невозможность оплаты, риск утечки данных, отсутствие гарантий доступности.
Решение: Создание отечественного on-premise (разворачиваемого локально) решения, которое гарантирует полный контроль над данными, работает внутри закрытого контура компании и обеспечивает качество связи на уровне мировых аналогов.

1.3. Целевая Аудитория

IT-компании и стартапы.

Государственные корпорации и Enterprise-сектор (требования 152-ФЗ).

Распределенные команды.

2. Ключевые Функциональные Возможности

2.1. Организация Пространства (Workspace)

Воркспейсы: Изолированные пространства для разных компаний или департаментов.

Иерархия: Воркспейс -> Категория -> Канал.

Ролевая модель: Владелец, Админ, Модератор, Участник, Гость (ограниченный доступ к 1 каналу).

2.2. Текстовые Коммуникации

Каналы: Публичные (доступны всем в воркспейсе) и Приватные (только по приглашению).

Треды (Threads): Ветки обсуждений внутри конкретного сообщения (критично для сохранения контекста).

Редактор: Поддержка Markdown, сниппетов кода с подсветкой синтаксиса, упоминаний (@user).

Реакции: Возможность ставить эмодзи на сообщения.

Статусы доставки: Отправлено / Доставлено / Прочитано.

2.3. Аудио и Видео (WebRTC)

Аудио-комнаты (Huddles): Постоянные голосовые каналы для быстрого обсуждения ("курилка").

Видеозвонки: Групповые звонки с поддержкой демонстрации экрана (Screen Sharing).

Качество: Адаптивный битрейт (SFU архитектура) для работы на слабом интернете.

Управление: Mute/Unmute, Push-to-Talk, управление правами голоса модератором.

2.4. Уникальные Фичи ("Киллер-фичи")

On-premise First: Поставка в виде Docker-контейнеров для развертывания на серверах заказчика одной командой.

Message-to-Task: Превращение любого сообщения в задачу (Тикет) с назначением ответственного и дедлайна в один клик.

AI Summary (План на v2.0): Автоматическая расшифровка звонков и генерация краткой выжимки (Summary) встречи.

3. Стек Технологий (Technology Stack)

Выбран на основе требований к высокой нагрузке (HighLoad) и независимости от вендоров.

Компонент                  Технология                     Обоснование

Backend                    Go (Golang)                    Высочайшая производительность, нативная поддержка многопоточности (goroutines), компиляция в один бинарный файл.

Frontend                   Vue 3 + Vite                   Современный реактивный фреймворк, высокая скорость рендеринга, легкая поддержка кода.

State Management           Pinia                          Управление состоянием интерфейса (списки чатов, статус юзера).

Styling                    Tailwind CSS                   Ускоренная верстка, гибкая кастомизация дизайна (Dark/Light mode).

Database                   PostgreSQL                     Надежное хранение реляционных данных (юзеры, сообщения, настройки).

Real-time                  WebSockets                     Протокол для мгновенной доставки сообщений и событий.

Calls (SFU)                LiveKit                        Open-source сервер для масштабируемых видеоконференций (лучшее решение на рынке).

Cache / MQ                 Redis                          Кэширование сессий, Pub/Sub для распределения сообщений между серверами.

File Storage               MinIO                          S3-совместимое хранилище файлов. Позволяет хранить файлы локально у клиента.

Mobile                     Flutter                        (В планах) Единая кодовая база для iOS и Android.

4. Архитектура Системы

Используется архитектура Modular Monolith (Модульный Монолит) на старте, с возможностью распила на микросервисы в будущем.

4.1. Компоненты

API Gateway (Nginx): Терминация SSL, раздача статики (Frontend), проксирование API запросов.

App Server (Go):

Auth Service: Регистрация, Login, JWT.

Chat Service: Обработка сообщений, история.

Presence Service: Статусы "Онлайн/Оффлайн".

Media Server (LiveKit): Обработка RTP потоков аудио/видео.

4.2. Безопасность

Все данные передаются по шифрованным каналам (HTTPS / WSS).

Пароли хранятся в виде хешей (bcrypt).

Доступ к API только по JWT токенам.

5. Требования к Инфраструктуре (Минимальные)

Для развертывания on-premise (до 500 пользователей):

CPU: 4 ядра

RAM: 8 GB

Disk: SSD 100 GB (зависит от объема пересылаемых файлов)

OS: Linux (Ubuntu/CentOS/Debian) с установленным Docker & Docker Compose.
